Glossary

Grain

What one row represents. Example: one customer, one order, one order item.

Fanout

A join that makes rows from a table repeat, usually one-to-many or many-to-many.

Metric inflation

An aggregate changes because repeated rows enter the aggregate. Commonly affects SUM and AVG.

Inflation-proof metric

Metric type stable under repeated rows in Lightdash's fanout logic: COUNT_DISTINCT, SUM_DISTINCT, AVERAGE_DISTINCT, MIN, MAX.

CTE

A named SQL scope in a WITH clause. Later scopes can read earlier scopes if they FROM/JOIN them.

PoP

Period-over-period. A generated comparison metric computed over a shifted time window.

Pivot long row

A row with pivot header values plus row_index and column_index. Not the final wide pivot row.

Anchor column

The pivot column whose value controls row ordering when sorting pivot rows by a metric.

Fanout decision checklist

  1. What is the starting/base table?
  2. What is the join relationship direction?
  3. Which table's rows can repeat in the joined result?
  4. Is the metric from a repeated table?
  5. Is the metric type vulnerable (SUM, AVG, plain COUNT) or inflation-proof/special?
  6. Does the repeated table have a primary key so Lightdash can build cte_keys_*?

Alias tracing algorithm

  1. Pick the final output alias.
  2. Search for AS "alias".
  3. If it appears more than once, identify which subsystem emitted each one.
  4. For the intended owner, list the source aliases in the expression.
  5. Verify the current scope reads from CTEs exposing those source aliases.
  6. Repeat until reaching base table columns or compiled metric SQL.

PoP invariant

current rows = base metric over current time window
previous rows = same base metric over shifted time window

join current to previous:
  non-time dimensions: null-safe equality
  comparison time dimension: current_time = previous_time + offset interval
  join type: LEFT JOIN, so current rows survive missing previous rows

Pivot pipeline cheat

original_query
  = MetricQueryBuilder output, with semantic metrics already decided

group_by_query
  = SELECT index cols + pivot header cols + aggregated value cols
    FROM original_query
    GROUP BY index cols + pivot header cols + helper dims

pivot_query
  = SELECT group_by_query fields
    + DENSE_RANK(...) AS row_index
    + DENSE_RANK(...) AS column_index

AsyncQueryService.runQueryAndTransformRows
  = stream ordered long rows
  = when row_index changes, flush current transformed row
  = value column name = valueField_aggregation + pivot value suffix
