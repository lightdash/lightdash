- `cy.login()` runs in `beforeEach` for every declared test (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:5-7`). It uses `cy.session` keyed by the seed admin email, POSTs `api/v1/login`, requires status 200, and validates reused sessions with `GET api/v1/user` (`packages/e2e/cypress/support/commands.ts:152-170`). Credentials are `demo@lightdash.com` / `demo_password!` (`packages/common/src/index.ts:474-481`). No UI login is exercised.
- `cy.scrollTreeToItem(text)` is used twice in test 1, once in tests 2 and 3, and three times in test 4 (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:12,25,28,72,112,158,165,172`). It finds `[data-testid="virtualized-tree-scroll-container"]` with a 10-second timeout, resets `scrollTop`, moves by half a viewport, waits 200 ms per step, and searches all descendants using exact/full/own-text heuristics. At the end it scrolls to the maximum and falls back to Testing Library `findByText` (`packages/e2e/cypress/support/commands.ts:773-823`). The application uses TanStack virtualization with five-row overscan and absolutely positioned items, explaining why ordinary `scrollIntoView` is insufficient (`packages/frontend/src/components/Explorer/ExploreTree/TableTree/Virtualization/VirtualizedTreeList.tsx:69-109`).
- `cy.getMonacoEditorText()` is called 6, 4, 4, and 5 times respectively (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:44-56,82-92,125-135,189-204`). Thus each SQL fragment repeats both the fixed delay and first-model assumption.
- `findByText`, `findByPlaceholderText`, `findByTestId`, and `findByRole` come from the global `@testing-library/cypress/add-commands` import (`packages/e2e/cypress/support/commands.ts:46`). The plugin registers every `find*` DOM Testing Library query as a Cypress retryable query and throws when a non-`All` query finds multiple elements (`packages/e2e/node_modules/@testing-library/cypress/dist/add-commands.js:5-12`; `packages/e2e/node_modules/@testing-library/cypress/dist/index.js:16-73`).

## Synchronization and timeout requirements

- `cy.session` must complete or validate before navigation. In a Playwright alternative, existing setup already performs the same login/user validation and persists admin storage state (`packages/e2e/playwright/auth.setup.ts:8-23`; `packages/e2e/playwright.config.ts:27-43`); no new auth helper is needed.
- Virtual-tree traversal is intentionally polling-like: 200 ms after every half-viewport move and another 200 ms at the end, under a 10-second container lookup (`packages/e2e/cypress/support/commands.ts:773-823`). A browser rewrite would need an explicit bounded scroll loop that waits for virtualizer rerenders, not ordinary `scrollIntoViewIfNeeded` on a currently absent node.
- The tests do not intercept query requests. Tests 1, 3, and 4 wait indirectly for result text after clicking Run query (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:31-37,115-119,179-183`). Test 2 has no result-ready assertion and immediately collapses cards after the click (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:75-80`), so its synchronization is weaker.
- SQL compilation is asynchronous (`useCompiledSql` POSTs `/api/v1/projects/{project}/explores/{table}/compileQuery`) and the helper's only explicit stabilization is 200 ms before reading the first Monaco model (`packages/frontend/src/hooks/useCompiledSql.ts:34-57,107-125`; `packages/e2e/cypress/support/commands.ts:751-769`). A replacement unit test should call QueryBuilder directly and eliminate this race.
- All four tests are independent after authentication and may run in any order. Within test 4, the source custom dimension must be created before its `Max` metric, then deselected only after the metric exists (`packages/e2e/cypress/e2e/app/customDimensions.cy.ts:142-177`). Preserve that semantic ordering in a focused unit input.
- Cypress command retries are per command and run-mode retries default to two (`packages/e2e/cypress.config.ts:12-21`). There is no explicit test-level timeout, debounce wait, aggregate polling deadline, or network response assertion in the source.
