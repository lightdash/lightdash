- `visit`, `wait`, `contains`, `request`, `get`, `within`, `find`, `click`, `type`, `intercept`, `as`, and `should` are Cypress built-ins. No other project custom command is invoked.

## State, seed, and environment assumptions

- The suite targets seed project `3675b69e-8324-4110-bdca-059031aa8da3`, `Jaffle shop` (`packages/common/src/index.ts:550-558`). It requires a cached or discoverable warehouse catalog containing nested schema `jaffle` and tables `customers` and `orders` (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:14-18,198-205,241-245`).
- The mutable variable named `schema` is actually the first top-level **database** key in `WarehouseTablesCatalog`; nested keys are schemas. The frontend performs the same transformation as `[database] = Object.keys(tablesData)` followed by nested schema entries (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:4,14-18`; `packages/frontend/src/features/sqlRunner/components/SqlEditor.tsx:76-100`). The port must preserve this distinction when building `"database"."jaffle"."table"` SQL.
- The suite setup visits project home, sleeps three seconds, opens `New`, follows the visible substring `SQL runner`, and separately GETs the catalog (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:6-21`). The actual menu item is titled `Query using SQL runner`; it is rendered only for users who can `manage` the project `SqlRunner` subject (`packages/frontend/src/components/NavBar/ExploreMenu.tsx:55-107`).
- The catalog request is GET `/api/v1/projects/{projectUuid}/sqlRunner/tables`; the source assumes success, assumes `body.results` is a nonempty object, and takes insertion-order key 0 without validating status/shape (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:14-19`). The controller requires an authenticated registered account (`packages/backend/src/controllers/sqlRunnerController.ts:58-80`), and the service requires `manage SqlRunner`, reads the user/project warehouse credentials, uses the cached catalog, or populates it from the warehouse if empty (`packages/backend/src/services/ProjectService/ProjectService.ts:7226-7269`).
- The SQL table picker itself also GETs that endpoint through React Query (`packages/frontend/src/features/sqlRunner/hooks/useTables.tsx:10-25`). Clicking `jaffle` expands a schema; clicking a table sets `SELECT * FROM "database"."schema"."table"` when SQL is empty or already a simple select (`packages/frontend/src/features/sqlRunner/components/Tables.tsx:61-99,178-249`).
- Each Run sends POST `/api/v2/projects/{projectUuid}/query/sql` with SQL, limit, parameters, and `invalidateCache: true`; the client polls GET `/api/v2/projects/{projectUuid}/query/{queryUuid}` with 250/500/1000 ms backoff until ready, then streams GET `/api/v2/projects/{projectUuid}/query/{queryUuid}/results` (`packages/frontend/src/features/sqlRunner/store/thunks.ts:39-66`; `packages/frontend/src/features/queryRunner/executeQuery.ts:12-79`). This requires the backend, application database, seed warehouse, async query execution, and result-stream/storage path. No third-party SaaS is logically required.
- Seed-data assumptions include `customers` fields that generate default chart labels/config around `created` and `age_sum`, plus `orders` fields `order_id`, `status`, and `customer_id` (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:205-206,231-268`). The tree renders tables only after local expansion and has a documented transient render-loop/unresponsive edge case (`packages/frontend/src/features/sqlRunner/components/Tables.tsx:178-225`). - The active test waits for a result header after each Run (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:205-206,244-245`). This is the correct semantic boundary, but Playwright should additionally pair the click with the POST `/api/v2/projects/{projectUuid}/query/sql` response and then use web-first table assertions; the backend operation remains asynchronous after the initial POST (`packages/frontend/src/features/queryRunner/executeQuery.ts:41-79`).

In Playwright, click the Monaco editor, use `ControlOrMeta+A`, insert the SQL, assert the editor text/value changed, and wait for Run to be enabled. Do not use a fixed one-second delay.
- The create workflow must wait for accessible spaces, initial-info Next, destination selection, POST success, and slug navigation before asserting view mode (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:211-226`; `packages/frontend/src/features/sqlRunner/components/SaveSqlChartModal.tsx:202-254`). Capture the create response so cleanup uses the returned UUID rather than a name lookup.

Wait for the chart view and expected labels, not merely the URL/title.

The repair dropdowns are portaled. - The skipped negative-query case is timing-sensitive by design: zero intercepted calls is sampled immediately after a local SVG update and observes only one endpoint (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:309-327`). This is another reason to move the invariant to a unit test rather than invent an arbitrary Playwright observation delay.
- Start with configured 10-second Playwright expectations, but allow focused query/chart readiness polling to reflect warehouse latency if evidence shows it is needed. Do not add global timeout changes or fixed sleeps.

## Locator and strictness risks

- `cy.contains('New')`, `cy.contains('SQL runner')`, `cy.contains('Save')`, `cy.contains('Run query')`, and `cy.contains('Edit chart')` are global substring queries (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:10-12,195,205,212,229,244,252,271`). Playwright strict mode can expose duplicate controls. Prefer roles and exact accessible names; scope Save/Next/Fix errors to the visible dialog when applicable. The SQL Runner menu's actual accessible text is `Query using SQL runner` (`packages/frontend/src/components/NavBar/ExploreMenu.tsx:80-105`).
- `cy.contains('jaffle')` and exact `customers`/`orders` searches are page-global (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:198-204,241-245`). Scope them to the tables sidebar/tree and use exact button text so Monaco, results, breadcrumbs, or hidden menu content cannot match.

## Nonstandard or surprising behavior

- The source calls its top-level catalog key `schema`, but it is the database name; `jaffle` is the schema (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:4,14-18,201-204`; `packages/frontend/src/features/sqlRunner/components/SqlEditor.tsx:82-100`).

This distinction matters when selecting the unit-test target.
- Successful SQL history is page-local Redux history and is appended only after a fulfilled query with nonempty rows/columns; it also clears `hasUnrunChanges` (`packages/frontend/src/features/sqlRunner/store/sqlRunnerSlice.ts:338-393`). It is not persistent server draft history.
- The active save relies on the default selected chart kind being vertical bar (`packages/frontend/src/features/sqlRunner/store/sqlRunnerSlice.ts:98-113`) and on generated initial fields including `created`/`age_sum`. Switching the SQL to `orders` intentionally leaves the stale chart config, producing the repair flow (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:231-268`).
- Saving silently chooses the first accessible space unless the user changes it (`packages/frontend/src/features/sqlRunner/components/SaveSqlChartModal.tsx:119-133,143-166`). The source never verifies which space owns the chart.
- The final update uses `customer_id` as Y and expects the generated aggregation label `Customer id avg`, while X expects display label `Status` (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:257-283`). These assertions depend on inferred field type/default aggregation behavior, not merely successful persistence.
- The chart-name duplication risk is subtler than a simple uniqueness failure: normal sequential reruns get generated slug suffixes, but same-name rows remain and concurrent slug generation is not locked around the read-then-insert sequence (`packages/backend/src/utils/SlugUtils.ts:57-73`; `packages/backend/src/database/migrations/20240704150503_add_saved_sql.ts:56-62`).
- The skipped no-extra-query test intercepts `/query/sql` only. Saved chart view uses `/query/sql-chart`, so this interception would not prove that all possible warehouse query endpoints stayed quiet (`packages/e2e/cypress/e2e/app/sqlRunner.cy.ts:309-327`; `packages/frontend/src/features/queryRunner/executeQuery.ts:126-151`).
- There are no downloads/uploads, popups, iframes, clipboard use, canvas assertions, drag-and-drop, virtualization, browser permissions, or timezone-sensitive behavior. Special mechanics are Monaco, Mantine portals, async query polling/streaming, ECharts SVG, fixed sleeps, and one skipped negative network alias.
